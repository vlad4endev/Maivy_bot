# Maivy Bot

Мультиплатформенный чат-бот для **Telegram** и **MAX** с **админ-панелью** для управления контентом, кнопками и пользователями.

## Сценарий

| Шаг | Что происходит |
|-----|----------------|
| **0** | Описание + фото профиля + ссылка на политику ПДн (до нажатия «Начать») |
| **/start** | Текст о Maivy → видео-кружочек → 4 кнопки меню |
| **1.1** | 7 экранов «Узнать больше» с кнопкой «Далее» |
| **1.2** | Ссылка на видео в Loom |
| **1.3** | Инструкция по поиску + скрипты + ссылка на Гростер |
| **1.4** | Контакт @daerit для внедрения |

## Быстрый старт

```bash
npm run install:all
cp .env.example .env
cp admin/.env.example admin/.env.local
# Заполните токены и ссылки
# Положите assets/welcome.jpg и assets/welcome-video.mp4
```

### 1. Запустите Convex (бэкенд админ-панели)

```bash
npx convex dev
```

В Convex Dashboard (или через CLI) задайте переменные окружения:

| Переменная | Назначение |
|---|---|
| `ADMIN_PASSWORD` | Пароль входа в админ-панель |
| `BOT_API_SECRET` | Секрет для связи бота с Convex |

### 2. Запустите админ-панель

```bash
npm run dev:admin
```

Откройте http://localhost:5173, войдите с паролем `ADMIN_PASSWORD`.

На странице **Боты** нажмите «Создать Maivy по умолчанию» — это загрузит все разделы и кнопки.

### 3. Запустите бота

В `.env` на сервере нужны только 3 переменные:

```env
CONVEX_URL=http://127.0.0.1:3210
BOT_API_SECRET=ваш-секрет
BOT_SLUG=maivy
```

Все остальное (токены, ссылки, медиа) — в админ-панели → **Настройки**.

```bash
npm run dev
```

## Админ-панель

| Раздел | Возможности |
|--------|-------------|
| **Дашборд** | Статистика, воронка, активность за 7 дней |
| **Боты** | CRUD ботов, платформы, токены, настройки ссылок |
| **Пользователи** | База пользователей TG/MAX, поиск, блокировка, экспорт CSV |
| **Разделы** | Редактирование текстов, предпросмотр HTML, черновики |
| **Кнопки** | Инлайн-клавиатуры, callback/URL, вкл/выкл |
| **Настройки** | Все параметры .env: токены, платформы, ссылки, медиа |
| **Аналитика** | Воронка конверсии, тепловая карта кнопок |
| **События** | Журнал start/callback действий |

Контент из Convex подгружается ботом автоматически (обновление каждые 60 сек). Если Convex недоступен — используется локальный fallback из `src/core/content.ts`.

## Настройка

### Токены

| Переменная | Где получить |
|---|---|
| `TELEGRAM_BOT_TOKEN` | [@BotFather](https://t.me/BotFather) |
| `MAX_BOT_TOKEN` | [business.max.ru](https://business.max.ru) |

### Контент и настройки

- **Админ-панель → Настройки**: токены, платформы, ссылки, медиа (вместо .env)
- **Админ-панель → Разделы / Кнопки**: тексты и клавиатуры
- **Fallback в коде**: `src/core/content.ts` (только если Convex недоступен)

### Политика ПДн

Шаблон: `docs/privacy-policy.md` — заполните поля, разместите на хостинге и укажите URL в `PRIVACY_POLICY_URL`.

### Медиа

| Файл | Назначение |
|------|------------|
| `assets/welcome.jpg` | Фото профиля бота |
| `assets/welcome-video.mp4` | Кружочек (TG) / видео (MAX) |

## Архитектура

```
convex/             — бэкенд (боты, пользователи, разделы, кнопки, аналитика)
admin/              — React админ-панель (Vite)
src/core/           — бизнес-логика бота (единая для обеих платформ)
src/platforms/      — адаптеры Telegram и MAX
src/lib/convex-client.ts — интеграция бота с Convex
```

## Production (Docker, без Convex login)

Полный стек на вашем сервере: PostgreSQL + Convex + админка + бот.

| Инструкция | Описание |
|------------|----------|
| **[docs/DOCKER.md](docs/DOCKER.md)** | Self-hosted Docker (рекомендуется) |
| **[docs/DEPLOY.md](docs/DEPLOY.md)** | Альтернатива: Convex Cloud |

```bash
cp .env.example .env
# Укажите PUBLIC_CONVEX_URL=http://ВАШ-IP:3210
./scripts/setup-self-hosted.sh
```

`npx convex login` **не нужен** — всё через Docker и `.env`.

### Только бот + облачный Convex

```bash
./scripts/docker-deploy.sh cloud
```

На сервере бота в `.env` нужны только `CONVEX_URL`, `BOT_API_SECRET` и `BOT_SLUG`. Токены и контент — в админ-панели.
